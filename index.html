Assign a custom domain name to your website

Step 1: Create an AWS account

Do this at Amazon Web Services.

Step 2: Register your domain

  1. Log into your AWS account and click on “Route 53” in the console under “Networking”.
  2. On the next page, click “Get Started Now” under “Domain Registration”.
  3. Click the button “Register Domain”.
  4. Choose a domain name and then select a suffix, like [.com] or [.io].

Wahey! You’re the now the proud owner of your very own custom domain name!

Step 3: Setting Up

We still have a few things to do before your domain name can work:

  1. Inform GitHub of the new domain name
  2. Set up DNS records in Route 53

Inform GitHub

  1. Open GitHub and access the repo: [your-user-name.github.io]
  2. Click the “New file” button.
  3. Call it [CNAME] and do not add an extension.

  4. Open the file and add your custom domain name to line 1:

  6. Under the title of the repo, click on “Settings”. Find the section “GitHub Pages” and check that there’s a message along the lines of:

Set up DNS records

  1. Access Route 53 on AWS
  2. Click on “Hosted Zones” - you’ll notice a Hosted Zone for your domain name. Open it.
  3. Confirm NS record:
    1. Notice the Hosted Zone already has an NS record with four values. Take a note of these.
    2. On the left, click “Registered domains” then click your domain name.
    3. On the right, look for the section titled “Name Servers”. These should be the same as what was in the NS record in the Hosted Zone.

  4. Create an A record:

    1. In your Hosted Zone, click the “Create Record Set” button. In the form, leave the “Name” field blank. Set the “Type” to [A - IPv4 address].
    2. Set “TTL (Seconds)” to [300]

    3. In the “Value” box, enter the following IP addresses:

      These belong to GitHub.

    4. Click “Save Record Set”.
  5. Create a CNAME record
    1. In your Hosted Zone, click the “Create Record Set” button.
    2. Set “Name” to [www]
    3. Set “Field” to [CNAME - Canonical name]
    4. Set “Value” to [your-user-name.github.io]
    5. Click “Save Record Set”.

Aaaand we’re done!

It should all be working now. If you go into your browser and enter your domain name into the URL you should be able to see your website.

Installation

Install

Icarus requires Hexo 3.0 and above.

Enable

  1. Rename themes\icarus\_config.yml.example to themes\icarus\_config.yml;
  2. Copy themes\icarus\_config.yml.site.example to your hexo blog’s root directory and rename it to _config.yml;
  3. Copy themes\icarus\_source\* into your hexo blog’s directory source;
  4. Then modify theme setting in _config.yml to icarus.

Update

Configuration

Theme configuration example

  • excerpt_link - Cooperate with <!-- more --> tag to show only part of the article in index pages.
  • fancybox - Enable Fancybox.
  • contacts - Your social network links, RSS link, etc.
  • widgets - Widgets displaying in sidebar.
  • thumbnail - Whether to show post thumbnails in the sidebar and archive pages.
  • links - Links displayed in the link widget.
  • google_analytics - Google Analytics ID.
  • favicon - Favicon path.

Site configuration example

  • author - Your name.
  • author_title - Title to your occupation.
  • avatar - Your avatar image link.
  • location - Where you live in.
  • disqus_shortname - Your Disqus shortname.

Features

Profile Sidebar

A nice place to show yourself. You can add your own information in your site’s _config.yml

Post Banner & Thumbnail

Thanks to atika, you can now add thumbnails and banners to every post to create better reading experience.

Responsive Layout

Icarus knows on what screen size you are browsering the website, and reorganize the layout to fit your device.

Custom Categories & Tags Pages

Get your categories and tags listed in single pages to make your blog more methodic.

Fancybox

Icarus uses Fancybox to showcase your photos. You can use Markdown syntax or fancybox tag plugin to add your photos.

Sidebar

Icarus provides 6 built-in widgets:

  • recent_posts
  • category
  • archives
  • tag
  • tagcloud
  • links

All of them are enabled by default. You can edit them in widget setting.
